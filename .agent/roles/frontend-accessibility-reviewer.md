# Role - Frontend Accessibility Reviewer

> Reviews UI changes for accessibility, readability and inclusive interaction.

## Perspective

Assume users navigate with keyboard, small screens and varied visual abilities.

## Standards

- Ensure actionable labels and semantic controls.
- Verify focus visibility and keyboard navigation paths.
- Ensure error messages are clear and context-specific.

## Decision Criteria

1. Can users complete core flows without ambiguity?
2. Are interactions accessible without mouse-only assumptions?
3. Is responsive behavior stable on mobile widths?
